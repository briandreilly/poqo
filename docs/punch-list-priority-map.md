# Punch List Priority Map

## Tier 1

Scope:
- core stability
- benchmark spine protection
- evaluator integrity
- profile discipline

Why it matters:
- this tier protects poqo's core claim that it makes the right next move and that its benchmark spine means something

Done enough means:
- core rules stay stable
- frozen benchmarks remain inspectable
- evaluator output stays trustworthy
- profiles do not corrupt routing

Do not overbuild:
- do not invent process layers, dashboards, or a benchmark management system

## Tier 2

Scope:
- router/interpreter tightening
- response quality tightening
- documentation lock

Why it matters:
- this tier improves judgment quality without destabilizing the benchmark spine

Done enough means:
- recurring edge cases are handled with small fixes
- response quality stays practical and compact
- core docs and historical reports are easy to navigate

Do not overbuild:
- do not turn heuristic cleanup into a rewrite or turn docs into a large internal handbook

## Tier 3

Scope:
- controlled real-world testing
- change control
- product readiness boundary

Why it matters:
- this tier controls how poqo leaves the lab without breaking the discipline that made the benchmark spine credible

Done enough means:
- real prompts are handled in a controlled way
- benchmark-affecting changes trigger the right reruns
- first-use boundaries are explicit

Do not overbuild:
- do not add feature sprawl, rollout complexity, or product infrastructure before controlled use justifies it
