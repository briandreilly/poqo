# Firm Enough

poqo is firm enough to stop engine polishing when all of these stay true at the same time:

- frozen benchmark packs remain stable across normal hardening changes
- the routing map changes only for repeated failures, not one-off taste tweaks
- calm, counter, and blunt stay clearly distinct while routing remains stable
- PROVE fires only for sufficiently defined, testable cases
- carried domain anchors stay earned and unrelated prompts drop them
- live harness runs stop surfacing repeated high-severity frame-hijack failures

When poqo is firm enough, stop adding engine categories and move effort to controlled real-world prompt packs, review passes, and product decisions outside the core judgment loop.
